About BatteryData

BatteryData is a data-driven battery materials research project, enabled by techniques from the field of natural language processing (NLP) and text mining.

Scientific battery research papers are the main data source of BatteryData, from which text-based information is retrieved and database is created.

Database Auto-generation

A battery materials property database was auto-generated using a bespoke battery version of ChemDataExtractor, a tool using state-of-the-art natural language processing algorithms to interpret language text that makes up the majority of scientific documents.

Database auto-generation methods include web-scraping, chemical-named entity recognition (CNER), and relationship extraction. A desktop graphical user interface(GUI) is provided to aid the use of this database and visualisation.

BatteryBERT Language Modelling

Huge pre-trained language models, including BatteryBERT, BatterySciBERT and BatteryOnlyBERT, are released for the text mining use. These models can be easily fine-tuned for down-stream tasks. All of the pre-trained and fine-tuned models can be downloaded from batterydata, which can be easily accessed by the Python package transformers.

Extractive Q&A System + Search Engine

The extractive Q&A system and search engine are built on Q&A fine-tuned BatteryBERT, in order to probe the database to give users a chance to ask dynamic questions from the data, and probe all knowledge in the literature about battery materials.

Document Classification

The document classifier is built on fine-tuned BatteryBERT for sequence classification, in order to classify battery or non-battery text given a certain paragraph from abstract of full text.
